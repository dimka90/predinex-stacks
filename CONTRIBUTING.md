# Contributing to Predinex Stacks

Thank you for your interest in contributing to Predinex! We welcome contributions from the community to help make this the best decentralized prediction market on Stacks.

## Code of Conduct

Please be respectful and professional in all interactions.

## How to Contribute

1.  **Find an Issue**: Browse the GitHub issues to find something you'd like to work on.
2.  **Fork and Branch**: Fork the repository and create a new branch for your feature or bug fix.
3.  **Implement Changes**: Make your changes, ensuring you follow the project's coding standards.
4.  **Test Your Changes**: Run `npm run test` to ensure all tests pass.
5.  **Submit a PR**: Create a Pull Request with a clear description of your changes.

## Coding Standards

- **Clarity**: Use clear, descriptive names for constants and functions. Follow the state-checks-effects pattern.
- **Frontend**: Use Tailwind CSS for styling and ensure components are responsive.
- **Documentation**: Update the README or other documentation files if your changes affect external APIs or workflows.

## Development Environment

- Node.js v16+
- Clarinet for smart contract development
- Next.js 16 (App Router)

For more details, see the [README.md](./README.md).
